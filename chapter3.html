Chapter 3: Computer Software and Operating System

Software - Introduction

Hardware by itself cannot perform any useful tasks without proper coordination with software. Software enables the hardware to function effectively and helps in mobilizing various resources. Software consists of sets of instructions known as programs, which tell the computer what to do. In essence, software is a collection of related programs and documentation. For any useful output to be produced, the hardware and software must work together. The different types of software include:

1. System Software

System software includes programs that control the operations of the computer system. These programs are not tailored to the specific needs of the user but are designed to assist in the management and operation of hardware resources. System software supports the running of other software, communicates with peripheral devices, and provides a platform for the development of other software.

Categories of System Software:

  • System Management: Responsible for managing the functioning of the computer system and performing management tasks between hardware and software.
    • Operating System: The first program loaded into the computer's memory when powered on. It manages the computer's hardware and software resources and provides an interface for users to interact with the system (e.g., Windows, Linux, macOS).
    • Utility Software: Supporting programs that perform specific tasks related to system maintenance, such as file compression or virus scanning (e.g., Norton Utility, WinZip).
    • Device Drivers: Software that allows the computer to communicate with external hardware devices like printers, scanners, or keyboards.
  • System Development: Helps programmers create new programs. System development software includes programming languages and language translators:
    • Programming Languages: Sets of instructions that allow users to write programs for the computer. These can be low-level or high-level languages.
    • Language Translators: Tools that translate programs written in one language into another (e.g., compilers, interpreters, assemblers).

2. Application Software

Application software refers to programs designed to help users perform specific tasks, such as word processing, spreadsheet management, or image editing. Unlike system software, application software directly addresses the needs of the user.

Types of Application Software:

  • Packaged Software: General-purpose software designed for wide usage but may not match every user's needs (e.g., Microsoft Office, Adobe Photoshop).
  • Customized Software: Tailored software designed to meet the specific needs of a user or organization (e.g., payroll systems, inventory management systems).

Operating System - Introduction

An operating system (OS) is a collection of programs that manage computer hardware and software resources. It provides a platform for application programs and acts as an intermediary between the user and the computer hardware. The main goal of an OS is to improve the efficiency and performance of a computer system.

Functions of an Operating System:

  • I/O Management: Manages the input and output of data between the computer and external devices such as keyboards, printers, etc.
  • User Interface (UI): Provides a means for users to interact with the computer. There are two primary types of UIs:
    • Character/Command User Interface (CUI): Used in systems like MS-DOS, where commands are typed.
    • Graphical User Interface (GUI): Found in systems like Windows and macOS, where users interact with icons and visual elements.
  • Security: Protects data and prevents unauthorized access by setting up user accounts and passwords.
  • Process Management: Allocates CPU time to various processes and ensures that they execute properly.
  • Memory Management: Manages the computer's memory (RAM), ensuring that each process has enough space to execute.
  • Data Management: Organizes data into logical groups such as files and directories.
  • Deadlock Prevention: Avoids situations where two or more processes are stuck in a state where they cannot proceed.
  • Time Sharing: Allows multiple users to access the computer system simultaneously.
  • Virtual Memory: Uses disk space as additional memory when the physical memory (RAM) is full.

Examples of operating systems include MS-DOS, Windows, macOS, Linux, etc.

Types of Operating Systems

A) On the Basis of Processing:

  • Multi-programming OS: Allows multiple programs to run simultaneously by sharing CPU time.
  • Multi-tasking OS: Allows multiple tasks or processes to run concurrently, typically used in single-user environments.
  • Multiprocessing OS: Uses more than one CPU to execute multiple processes simultaneously.
  • Time-sharing OS: Shares processor time among multiple users, providing the illusion of simultaneous execution.
  • Batch Processing OS: Executes jobs in batches without user interaction during processing.
  • Multi-threading OS: Divides a process into smaller threads that are executed concurrently.
  • Real-time OS: Processes data within a fixed time, typically used in systems where time constraints are critical.
  • Online Processing OS: Processes transactions as soon as they occur, with immediate feedback to the user.

B) On the Basis of User Interface (UI):

  • Character User Interface (CUI): A command-based interface where users interact by typing text commands (e.g., MS-DOS).
  • Graphical User Interface (GUI): A visual-based interface where users interact with icons and visual elements (e.g., Windows, macOS).

Graphical User Interface (GUI) vs Character User Interface (CUI) and Operating Systems

Features of Graphical User Interface (GUI)

  • It is user-friendly, making it easy to use.
  • The user doesn't have to remember commands or syntax.
  • GUI supports full multimedia and animation.
  • It requires larger memory for its processing.
  • GUI environments support multimedia and animation.
  • It can be used in network environments.
  • GUI is multi-tasking, multi-programming, and multi-user operating system.
  • Every type of peripheral device can be used with GUI.

Basic Components of GUI:

  • Icon: Icons are small pictures representing commands, files, or windows.
  • Pointer: A symbol (usually an angled arrow) controlled by a pointing device such as a mouse, trackpad, or light pen.
  • Desktop: The area on the display screen where icons are grouped, referred to as the desktop.
  • Windows: Windows can be resized, moved, and allow different programs or information to be displayed.
  • Menus: Allow users to select commands from a list of options.
  • Folders: A graphical representation of a directory or collection of files.

Operating Systems Based on User Mode

1. Single User OS

A single user OS is designed for use by a single user at a time. There are two types:

  • Single User, Single Tasking: Can only handle one task at a time (e.g., MS-DOS).
  • Single User, Multi-tasking: Can handle multiple tasks, allowing the user to perform more than one action simultaneously.

2. Multi User OS

A multi-user OS allows multiple users to access the system at the same time through different terminals. It shares computer resources among users, giving them the impression of simultaneous service, even though they are being served one at a time. This is known as time-sharing.

  • Examples: Linux, UNIX

Some Important Operating Systems in Use

Open Source Operating Systems

Open source operating systems have source code that is freely available and can be modified by anyone. Users do not need to buy a license to use the software, and they can redistribute the modified version legally.

  • Examples: Linux, Unix, Minix, Open Solaris

Unix

UNIX is a powerful, flexible, and multi-tasking OS developed by AT&T Bell Labs in 1970. It is used in larger machines like mainframes and mini-computers. It has influenced many operating systems such as Linux and Solaris.

Linux

Linux is an open-source UNIX-based OS that can run on various platforms. It was developed by Linus Torvalds and released as a free, user-developed project. Linux is popular for servers and distributed through different distributions like Red Hat, Ubuntu, and Fedora.

Mac OS

Mac OS is a GUI-based operating system used in Apple Macintosh computers. It is known for its high-quality graphical user interface.

Some Terminologies Related to Operating Systems

Spooling

Spooling is used to balance the speed differences between a fast CPU and slower input/output devices. It allows multiple files to be queued and printed sequentially, while the CPU remains free for other tasks.

Process

A process is a program in execution. It can be in different states such as running, ready, or blocked. A process is dynamic, whereas a program is static.

Thread

Threading is a technique used in operating systems to divide a process into smaller sub-processes that can run concurrently, improving efficiency and minimizing the cost of context switching.

Scheduling

Scheduling refers to the process of deciding which process should run first when multiple processes are competing for CPU time. Two types of scheduling are:

  • Priority Scheduling: Jobs are executed based on priority, with high-priority jobs being executed first.
  • Time-sharing Scheduling: Each process is given a fixed amount of time to execute.

Buffering

Buffering uses small storage devices to hold data during input/output operations, improving the efficiency of the system. This is particularly useful in applications like audio/video streaming.

Virtual Memory

Virtual memory allows an OS to use space from secondary memory when primary memory (RAM) is insufficient. It divides large programs into smaller chunks, known as pages, and stores them temporarily in secondary memory.
